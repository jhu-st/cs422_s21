HW2

caution
  • You are expected to work individually.
  • Due: Tuesday March 9th at 11pm EST (Baltimore time).
  • This assignment is worth 50 points.
Objectives
  • Coverage criteria and submsuption
  • Graph-based testing/coverage criteria
  • Logic-based testing/coverage criteria
  • Code coveragle tool (JaCoCo)

Task 1#

info

Answer each of the following questions clearly and concisely. Note that some of the essay questions are intentionally vague and/or open-ended to stimulate your thinking. Answer these questions in your own words the best way you can.

  1. Name at least three advantages of coverage-criteria-based testing (i.e., utilizing coverage-criteria in testing). Also, name two downsides of it.

  2. Subsumption has a significant weakness. Suppose criterion C1 subsumes criterion C2 and that test set T1 satisfies C1 and test set T2 satisfies C2. It is not necessarily the case that T2 is a subset of T1. It is also not necessarily the case that T1 reveals a fault if T2 reveals that fault. Give actual examples of these two cases.

  3. How else could we compare coverage criteria besides subsumption?

  4. Write a predicate and a set of additional constraints so that correlative active clause coverage (CACC) is infeasible with respect to some clause, but general active clause coverage (GACC) is feasible.

Deliverables

The deliverables for this task is your answers to the questions above saved in a document named task1 e.g. task1.pdf.

Task 2#

Download Primes class. Use the method getPrimes() for questions below:

  1. Draw the control flow graph for the getPrimes() method.
  2. Consider test cases t1 = (n = 3) and t2 = (n = 5). Although these tour the same prime paths in getPrimes(), they do not necessarily find the same faults. Design a simple fault that t2 would be more likely to discover than t1 would.
  3. For getPrimes(), find a test case such that the corresponding test path visits the edge that connects the beginning of the while statement to the for return statement without going through the body of the while loop.
  4. List the test requirements for Node Coverage, Edge Coverage, and Prime Path Coverage.
  5. List test paths that achieve Node Coverage but not Edge Coverage on the graph.
  6. List test paths that achieve Edge Coverage but not Prime Path Coverage on the graph.
  7. List test paths that achieve Prime Path Coverage on the graph.
  8. Implement (in JUnit) test cases to cover all test paths listed in the previous step to achieve Prime Path Coverage.
  9. Run Jacoco to generate a report by running your JUnit tests against getPrimes() and confirm 100% edge coverage is achieved.
info

If it is not possible to achieve/accomplish any of the above items, make sure you mention that in your answer with a detailed explanation as to why.

Deliverables

The deliverables for this task are 1) your answers to questions 1-7 saved in a document named task2 e.g. task2.pdf, 2) A test class named PrimesTest.java that contains your answers to part 8, and 3) The generated Jacoco report compressed as a single .zip file named jacoco_report.zip.

Task 3#

Method isXnYValid(int x, int y, int a, int b); takes four int values x, y, a, and b as inputs and returns true if all of the following hold:

  • x is even and is in the range of [a, b]
  • y is outside the range of (a, b)
  • x is greater than y, or x and y are not equal unless x is a perfect square number

The isXnYValid method has a precondition a <= b. In other words, you may assume that anytime isXnYValid is invoked, it is always the case that a <= b. You may also assume there exists a boolean function named isPerfectSquare(int x) that you can call.

  1. Formulate all the above as one logic predicate
  2. Write both test requirements and concrete test cases to achieve predicate coverage, but not clause coverage.
  3. Write both test requirements and concrete test cases to achieve clause coverage, but not predicate coverage.
  4. Write both test requirements and concrete test cases to achieve correlative active clause coverage (CACC).
Hint

when working on (3), first try to get your head around what that statement really means for x and y as integer values and then try to formulate it in the simplest form possible (i.e., using the least number of clauses).

info

If it is not possible to achieve/accomplish any of the above items, make sure you mention that in your answer with a detailed explanation as to why.

Deliverables

The deliverables for this task is your answers to the questions above saved in a document named task3 e.g. task3.pdf.

For each task, make sure to submit all the necessary deliverables including JUnit Test classes (.Java files NOT .class files) and documents.
